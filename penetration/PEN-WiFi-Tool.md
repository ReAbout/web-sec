# 近源渗透硬件与随身 Wi-Fi 改造

## 0x01 概述

近源场景的重点不是高性能，而是低功耗、隐蔽、可远程维护。常见目标：

- 挂在目标附近，持续回连。
- 作为网关、代理或扫描节点。
- 需要 4G / Wi-Fi / USB 供电能力。

## 0x02 设备选择

优先考虑以下特性：

- 芯片方案资料多，社区活跃。
- 能刷 OpenWrt 或其他可控系统。
- 支持 USB 供电，便于长期部署。
- 至少具备一个稳定的上网链路。

常见方向：

- 高通 410 / 8916 系列随身 Wi-Fi
- 便携路由器
- 树莓派或同类单板

## 0x03 随身 Wi-Fi 改造思路

准备：

- 高通 410 芯片板子
- TTL / ADB / Fastboot 等刷机入口
- 对应固件和回刷方案

基础流程：

1. 识别设备型号和 SoC。
2. 确认是否已有现成 OpenWrt 固件。
3. 备份原厂分区和关键配置。
4. 刷入固件并验证网络、SSH、持久化能力。
5. 补齐代理、扫描、隧道工具。

## 0x04 常见用途

### 1. 代理网关

把设备作为近源网关，向外打隧道，内部终端流量统一转发。

### 2. 扫描节点

适合做轻量资产发现、存活探测和服务识别，避免让主攻击机直接暴露在目标网络中。

### 3. 中继节点

常与 `ssh -D`、`frp`、`chisel`、`socat`、`tun2socks` 组合使用。

## 0x05 设备上线后的基础加固

- 第一时间修改默认口令。
- 关闭不必要的管理面。
- 固定回连方式和管理地址。
- 配置 watchdog，避免离线后无法恢复。
- 保留原厂固件和回刷文档。

## 0x06 注意事项

- 先确认供电是否稳定，很多“掉线”其实是电源问题。
- 量产型随身 Wi-Fi 的散热和闪存寿命通常一般，不适合长期高负载。
- 刷机前一定先确认恢复手段，避免直接刷成砖。
